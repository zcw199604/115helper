# 背景
当前 `115helper` 已在 115 Open 上传主链路上部分对齐 MoviePilot-Plugins，但目录上传主流程仍保留了面向批量同步优化的实现：先收集叶子目录并预创建、再同步远端目录文件列表、最后按目录批次执行防重与上传。

现有实现与插件存在三类关键差异：
- 目录不存在时，`115helper` 倾向于通过 `fs_dir_getid + fs_makedirs_app` 一次性确保完整路径存在；插件则倾向于按目标文件路径从根目录逐级 `list_files → create_folder`。
- `115helper` 在上传前会同步整个远端目录文件列表；插件目录上传主流程则不先同步整目录，而是在上传后轮询单文件结果。
- `115helper` 当前目录创建与远端缓存准备强绑定在 `_get_folder/prepare_dir_context` 这一套批处理语义中，不利于按插件方式精确复刻目录上传顺序。

# 目标
将当前项目的“本地目录上传到 115”的执行方式尽可能调整为与 MoviePilot-Plugins 一致，重点对齐：
- 目录不存在时的逐级探测与逐级建目录方式；
- 单文件上传前的目录准备顺序；
- 上传完成后的结果确认方式；
- 对外可观察行为（日志、异常语义、调用时序）。

# 非目标
- 不要求完全复刻插件中的所有外围能力（通知、监控、第三方中心服务、上传等待策略）。
- 不在本次方案中改动前端交互结构，除非后端配置项变化必须同步暴露。
- 不强制移除现有 Open 上传链路；若其顺序已与插件一致，可保留并复用。

# 成功标准
1. 远端目录准备流程可按插件方式执行：优先查父目录，不存在时从根目录逐级列目录并创建缺失目录。
2. 主同步流程不再依赖“预创建叶子目录 + 上传前同步整个目录文件列表”作为默认路径。
3. 上传后增加单文件级轮询确认，使执行顺序与插件目录上传语义保持一致。
4. 现有秒传 / Open 上传能力不回退，且目录对齐改造不破坏取消控制、日志记录和结果落库。
5. 知识库明确记录新旧流程差异、兼容策略与回退方案。
