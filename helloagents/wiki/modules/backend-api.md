# backend-api

## 目的
提供前端控制台所需的配置管理、任务执行与运行查询接口。

## 模块概述
- **职责:** 暴露 REST API、参数校验、统一响应、错误处理、任务触发。
- **状态:** 🚧开发中
- **最后更新:** 2026-03-30

## 规范
- API 只负责任务编排和状态返回，不直接承载重型同步逻辑。
- 已实现同步任务列表/详情/创建/更新/删除/启停，以及运行列表/详情/日志/实时日志流/重试/取消和系统设置接口。
- 手动执行与重试接口现已立即返回，并在后台线程中继续执行任务。
- 后端可直接托管 `frontend/dist` 静态资源，支持单容器对外提供前后端服务。
- 仓库已提供 GitHub Actions 镜像构建工作流，可自动打包并推送 Docker Hub。
- 任务列表返回调度状态快照，包括最近执行、下次执行与最近状态。
- 手动执行接口已接入同步执行服务，并对同一同步任务做并发保护。
- 敏感配置读取时仅返回是否已配置，不返回明文 Cookie。


#### [GET] /api/v1/runs/{run_id}/logs
**描述:** 获取某次任务执行的阶段日志。

#### [POST] /api/v1/tasks/{task_id}/toggle
**描述:** 启用或停用某个同步任务。


#### [GET] /api/v1/runs/{run_id}/logs/stream
**描述:** 通过 SSE 实时订阅某次运行的新增日志。
